# Community Strategy Reference

Use this reference when designing the member promise, growth loop, value model,
or community-builder strategy.

## Best-Builder Principles

- Start from member alignment before platform.
- Pick one primary value pillar before optimizing metrics.
- Make the ritual simple enough to repeat weekly.
- Reward contribution, proof, and helpfulness over message volume.
- Create a ladder from visitor to member to contributor to host.
- Keep recognition opt-in and non-shaming.

## Starlight Translation

| Strategy Concern | Starlight Answer |
|---|---|
| Identity | Builder in a creation cell |
| Belonging | 3-5 person remembered group |
| Progress | Weekly proof object |
| Contribution | Help one peer unblock or sharpen |
| Retention | Friday closure plus next commitment |
| Growth | Approved artifacts and member stories |
| Measurement | Commitment, proof, reflection, helpfulness |

## Value Pillars

- `support`: unblock members faster.
- `product`: discover problems and workflow improvements.
- `acquisition`: create qualified referrals and invitations.
- `content`: package member artifacts into approved stories.
- `engagement`: sustain weekly participation and contribution.
- `success`: help members achieve visible progress.

Choose one as primary for each pilot. Do not optimize all at once.
