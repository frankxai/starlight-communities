# Community Strategy

Starlight Communities borrows the best durable ideas from community builders
and translates them into agentic operations.

## Community Canvas Translation

| Canvas Area | Starlight Translation |
|---|---|
| Identity | Member is a builder in a creation cell |
| Experience | Monday-Friday quest loop |
| Structure | 3-5 person cells, proof, reflection, memory |

## SPACES Translation

Pick one primary value pillar per pilot:

- `support`: unblock members faster.
- `product`: discover member problems and workflow improvements.
- `acquisition`: generate qualified referrals and invitations.
- `content`: package approved member artifacts into stories.
- `engagement`: sustain weekly participation.
- `success`: help members achieve visible progress.

The primary pillar determines metrics. Metrics do not determine strategy.

## Member Journey

```mermaid
flowchart LR
  Visitor --> NewMember
  NewMember --> CellMember
  CellMember --> Contributor
  Contributor --> Host
  Host --> Alumni
  Alumni --> Visitor
```

| Stage | Proof |
|---|---|
| Visitor | RSVP or explicit no |
| New Member | Intake and first commitment |
| Cell Member | Thursday proof object |
| Contributor | Peer-help note |
| Host | Reusable ritual card |
| Alumni | Final reflection and next open loop |

## Anti-Patterns

- Platform before promise.
- Leaderboard before trust.
- Content calendar before proof.
- Over-automation before consent.
- Public recognition without opt-in.
- Raw message volume as a primary metric.
