# Integration Strategy

Do not integrate a platform until the concierge loop works.

## Recommended Order

1. Email or existing private channel for pilot prompts.
2. SIS-style private memory for canonical records.
3. Circle if polished paid community experience matters.
4. Discord if voice, stage, forum, and high-social energy matter.
5. Slack if professional private-team operations matter.
6. Native FrankX/Starlight UI after retention and artifact output are proven.

## Adapter Rules

- Every adapter starts in dry-run mode.
- Every external send is human-gated.
- Every platform event maps back to a canonical object.
- Platform IDs are shadow references, not primary IDs.
- No adapter stores secrets in repo.

## Platform Fit

| Platform | Best Use | Warning |
|---|---|---|
| Circle | polished community, courses, embedded spaces | higher-tier APIs; do not make it canonical |
| Discord | voice, stage, forum, fast social presence | noisy by default; needs strong channel discipline |
| Slack | professional cells and ops workflows | less magical for creator community energy |
| Skool | creator-course/community shell | public automation surface is not the canonical spine |
| Native web | high-trust custom member/admin UX | only worth building after pilot proof |
